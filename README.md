## 기능 구현 목록

구매자의 할인 혜택과 재고 상황을 고려하여 최종 결제 금액을 계산하고 안내하는 결제 시스템

### 유틸리티
[x] products.md로 부터 모든 상품의 재고 정보를 읽어 String 리스트 형태로 반환할 수 있다.   
[x] promotions.md로 부터 모든 상품의 재고 정보를 읽어 String 리스트 형태로 반환할 수 있다.   
[x] String 리스트를 쉼표로 구분하여 Map<String, String> 리스트 형태로 변환하여 반환할 수 있다.   
[x] String을 int형으로 변환할 수 있다.   
[x] String을 LocalDate 형으로 변환할 수 있다.   
[] 업데이트된 재고를 products.md에 업데이트 할 수 있다.   

### 상품
[x] 상품 이름, 가격, 재고, 프로모션 정보를 저장할 수 있다.  
[] 상품 이름으로 특정 상품의 정보를 가져올 수 있다.
[] 각 상품은 고객이 구매할 때마다, 결제된 수량만큼 재고에서 차감되며 업데이트 될 수 있다.
[] 현재 가지고 있는 모든 상품 정보를 가져올 수 있다.

### 프로모션
[x] 프로모션 이름, 프로모션을 위해 구매해야할 상품 개수, 증정 개수, 프로모션 시작기간, 종료기간, 활성화 상태를 저장할 수 있다.      
[x] 프로모션 이름으로 프로모션 정보를 조회할 수 있다.   
[] 상품 이름과 구매 개수로 프로모션 적용 여부를 조회할 수 있다.   
[] 프로모션 적용이 가능한 상품에 대해 요청 수량이 불충분한 경우, 추가적으로 필요한 수량을 반환할 수 있다.   
[] 프로모션 재고가 부족하여 프로모션 혜택 없이 결제해야 하는 일부 수량을 반환할 수 있다.   
[] 현재 날짜가 프로모션 적용 기간 내에 포함되어야 프로모션 적용이 가능하다.   

### 주문
[] 주문 아이템 리스트를 저장할 수 있다.   
[] 주문 아이템 리스트를 조회할 수 있다.   
[] 영수증 출력 후 추가 구매를 진행할지 또는 종료할지를 선택할 수 있다.
[] 총 구매액을 계산할 수 있다.   
[] 프로모션 및 멤버십 할인을 반영하여 최종 결제 금액을 계산할 수 있다.   

### 주문 아이템
[] 상품의 이름과 구매 수량으로 주문 아이템을 추가할 수 있다.   

### 멤버십 할인
[] 멤버십 할인은 프로모션이 적용되지 않은 상품들 총 금액의 30%를 할인시킨다.   
[] 멤버십 할인은 프로모션 적용 후 남은 금액에 대해 적용한다.   
[] 멤버십 할인의 최대 한도는 8,000원이다.

### 입력
[] 구매할 상품과 수량을 [상품명-수량] 형식으로 입력받는다.
[] 쉼표(,) 각 상품을 구분한다.
```
[콜라-10],[사이다-3]
```

[] 프로모션 적용이 가능한 상품에 대해 고객이 해당 수량보다 적게 가져온 경우, 그 수량만큼 추가 여부를 입력받는다.   
- Y: 증정 받을 수 있는 상품을 추가한다.   
- N: 증정 받을 수 있는 상품을 추가하지 않는다.   

[] 프로모션 재고가 부족하여 일부 수량을 프로모션 혜택 없이 결제해야 하는 경우, 일부 수량에 대해 정가로 결제할지 여부를 입력받는다.
- Y: 일부 수량에 대해 정가로 결제한다.
- N: 정가로 결제해야하는 수량만큼 제외한 후 결제를 진행한다.

[] 멤버십 할인 적용 여부를 입력 받는다.   
- Y: 멤버십 할인을 적용한다.
- N: 멤버십 할인을 적용하지 않는다.

[] 추가 구매 여부를 입력 받는다.
- Y: 재고가 업데이트된 상품 목록을 확인 후 추가로 구매를 진행한다.
- N: 구매를 종료한다.

### 영수증 출력
[] 영수증은 구매한 상품명, 수량, 가격을 포함한다.   
[] 영수증은 프로모션에 따라 무료로 제공된 증정 상품의 목록을 포함한다.   
[] 영수증은 아래 내용을 포함한다.
- 총구매액: 구매한 상품의 총 수량과 총 금액
- 행사할인: 프로모션에 의해 할인된 금액
- 멤버십할인: 멤버십에 의해 추가로 할인된 금액
- 내실돈: 최종 결제 금액

### 예외
[] 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`를 발생시킨다.   
[] 예외 발생 시 "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.   
[] 예외는 `IllegalArgumentException`, `IllegalStateException` 등과 같은 명확한 유형을 처리한다.   
[] 각 상황이 발생 시 아래의 에러 메시지를 출력한다.   
- 구매할 상품과 수량 형식이 올바르지 않은 경우: `[ERROR] 올바르지 않은 형식으로 입력했습니다. 다시 입력해 주세요.`
- 존재하지 않는 상품을 입력한 경우: `[ERROR] 존재하지 않는 상품입니다. 다시 입력해 주세요.`
- 존재하지 않는 프로모션 조회를 시도한 경우: `[ERROR] 존재하지 않는 프로모션입니다.`
- 구매 수량이 재고 수량을 초과한 경우: `[ERROR] 재고 수량을 초과하여 구매할 수 없습니다. 다시 입력해 주세요.`
- 기타 잘못된 입력의 경우: `[ERROR] 잘못된 입력입니다. 다시 입력해 주세요.`
- 파일을 불러올 수 없는 경우: `[ERROR] 파일을 불러올 수 없습니다. 경로를 확인해 주세요.`
- 상품의 형식이 잘못된 경우: `[ERROR] 잘못된 상품입니다.`
- 프로모션의 형식이 잘못된 경우: `[ERROR] 잘못된 프로모션입니다.`
- 판매 상품이 비어있는 경우(재고 소진): `[ERROR] 재고가 모두 소진되었습니다. 판매 상품을 채워 주세요.`
