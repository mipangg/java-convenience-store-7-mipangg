# 미션 4. 편의점

## 기능 구현 목록
- 재고 개별 상품 관리
  - [x]상품명, 가격, 재고 수량, 프로모션 타입을 저장한다.
  - [x]원하는 상품에 대한 재고 정보를 알려준다.
  - [x]구매 후 상품의 재고를 차감한다.

- 재고 관리
  - [x]products.md 파일을 이용하여 상품 등록한다.
  - [x]상품명으로 재고를 조회한다.
  - [x]없는 상품명으로 재고 조회를 시도할 경우 예외를 발생 시킨다.
  - [x]재고가 충분한지 확인한다.
  - [x]재고가 충분할 경우 구매 후 재고를 차감한다.
  - [x]재고가 충분하지 않으면 재고가 부족함을 알린다.

- 구매 아이템
  - [x]상품과 수량을 저장한다.
  - [x]상품 가격을 계산한다.

- 구매 처리
  - [x]유저가 주문한 항목 리스트를 저장한다.
  - 상품별 가격과 수량을 곱한 후, 프로모션 및 멤버십 할인 정책을 반영하여 최종 결제 금액을 계산한다.
  - 영수증을 구성한다.

- 프로모션 할인
  - [x]promotions.md 파일을 이용하여 프로모션 정보를 얻는다.
  - [x]프로모션 제품인지 확인한다.
  - [x]구매하고 싶은 수량 만큼 프로모션 상품의 재고가 충분한지 확인한다.
  - 프로모션 재고가 부족하여 일부 수량을 프로모션 혜택 없이 결제해야 하는 경우, 일부 수량에 대해 정가로 결제할지 묻는다.
  - 프로모션 재고를 우선적으로 차감하며, 부족할 경우 일반 재고에서 차감한다.
  - [x]프로모션 적용 기준에 맞는 수량인지 확인한다.
  - 프로모션 적용이 가능한 상품에 대해 고객이 해당 수량보다 적게 가져온 경우, 필요한 수량을 추가로 가져올지 묻는다.
  - [x]프로모션 할인 금액을 계산하고 프로모션 재고를 차감한다.

- 멤버십 할인
  - 회원은 프로모션 미적용 금액의 30%를 할인받는다.
  - 프로모션 적용 후 남은 금액에 대해 멤버십 할인을 적용한다.
  - 멤버십 할인의 최대 한도는 8,000원이다.

- 추가 구매
  - 유저가 추가 구매를 원할 경우 재고 목록을 업데이트하여 사용한다.

- 입력 처리
  - 상품명과 수량을 사용자로부터 입력 받는다.
  - 프로모션 혜택 적용 여부를 입력 받는다.
  - 멤버십 할인 적용 여부를 입력 받는다.
  - 추가 구매 여부를 입력 받는다.
  - 잘못된 입력이 발생할 경우 다시 입력 받는다.

- 출력 처리
  - 환영 인사와 함께 상품명, 가격, 프로모션 이름, 재고를 안내한다.
  - 프로모션 적용이 가능한 상품에 대해 고객이 해당 수량만큼 가져오지 않았을 경우, 혜택에 대한 안내 메시지를 출력한다.
    - 현재 {상품명}은(는) 1개를 무료로 더 받을 수 있습니다. 추가하시겠습니까? (Y/N)
  - 프로모션 재고가 부족하여 일부 수량을 프로모션 혜택 없이 결제해야 하는 경우, 일부 수량에 대해 정가로 결제할지 여부에 대한 안내 메시지를 출력한다.
    - 현재 {상품명} {수량}개는 프로모션 할인이 적용되지 않습니다. 그래도 구매하시겠습니까? (Y/N)
  - 멤버십 할인 적용 여부를 확인하기 위해 안내 문구를 출력한다.
    - 멤버십 할인을 받으시겠습니까? (Y/N)
  - 영수증을 출력한다.
    - 영수증은 구매 상품 내역, 증정 상품 내역, 총구매액, 행사할인, 멤버십할인, 내실돈을 출력한다.
  - 추가 구매 여부를 확인하기 위해 안내 문구를 출력한다.
    - 추가 구매 여부를 확인하기 위해 안내 문구를 출력한다.
  - 잘못된 입력 처리는 오류 메시지로 안내한다.

- 예외 처리
  - 구매할 상품과 수량 형식이 올바르지 않은 경우: [ERROR] 올바르지 않은 형식으로 입력했습니다. 다시 입력해 주세요.
  - 존재하지 않는 상품을 입력한 경우: [ERROR] 존재하지 않는 상품입니다. 다시 입력해 주세요.
  - 구매 수량이 재고 수량을 초과한 경우: [ERROR] 재고 수량을 초과하여 구매할 수 없습니다. 다시 입력해 주세요.
  - 기타 잘못된 입력의 경우: [ERROR] 잘못된 입력입니다. 다시 입력해 주세요.
